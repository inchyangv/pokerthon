## P0: 대회 진행 불가 (Must Fix)

### 1. 어드민 수동 핸드 시작 엔드포인트 없음
- **현상**: 핸드를 시작하는 방법이 bot_runner의 `_auto_start_hands()`뿐. 어드민이 수동으로 핸드를 시작할 API/UI가 없음.
- **영향**: `BOT_ENABLED=false`면 핸드 시작 자체가 불가능. 첫 핸드도, 이후 핸드도 어드민이 직접 트리거할 수 없음.
- **파일**: `app/bots/runner.py:177-216` (유일한 핸드 시작 로직), `app/services/hand_completion.py:127` (`_delayed_next_hand`는 핸드 완료 후 자동 트리거)
- **해결**:
  - `POST /admin/tables/{table_no}/start-hand` 엔드포인트 추가
  - 어드민 테이블 상세 UI에 "핸드 시작" 버튼 추가
  - `_delayed_next_hand` 자동 트리거는 **비활성화** (어드민이 매 핸드 수동 시작) 또는 **유지** (첫 핸드만 수동, 이후 자동) — 운영 방식에 따라 결정

### 2. 토너먼트 우승자 판정 없음
- **현상**: 테이블에 1명만 남으면 핸드가 안 시작될 뿐(2명 미만 체크), 우승 선언/알림 없음.
- **영향**: 대회 종료를 어드민이 수동으로 확인해야 함. 관전 화면에도 우승자 표시 없음.
- **파일**: `app/services/hand_completion.py:118-124`
- **해결**: 핸드 완료 후 eligible 1명이면 → 테이블 자동 PAUSED + 우승자 로깅 + 뷰어에 우승자 표시.

### 3. 블라인드 에스컬레이션 없음
- **현상**: SB/BB가 테이블 생성 시 고정 (기본 1/2). 40칩으로 1/2 고정이면 게임이 매우 길어짐.
- **영향**: 토너먼트가 실질적으로 끝나지 않거나 매우 오래 걸림.
- **파일**: `app/config.py:11-12`, `app/services/hand_service.py:150-151` (`settings.SMALL_BLIND/BIG_BLIND` 하드코딩)
- **해결**: 블라인드 레벨 스케줄 도입 (시간 기반). 예: 15분마다 1/2 → 2/4 → 3/6 → 5/10.
  - Table 모델에 `blind_level` 필드 또는 별도 스케줄 테이블
  - `start_hand`에서 현재 블라인드 레벨 조회하여 적용
  - 어드민 UI에서 현재 블라인드 레벨 표시 + 수동 변경 가능

### 4. 10명 참가 시 테이블 머징
- **현상**: 10명이면 2테이블 필요 (max 9석). 탈락으로 9명 이하가 되면 1테이블로 합쳐야 함. 테이블 간 플레이어 이동 로직 없음.
- **영향**: 한 테이블에 1명만 남으면 게임 불가.
- **파일**: 관련 로직 자체가 부재
- **해결**: 어드민 수동 머징 기능:
  - `POST /admin/tables/{table_no}/move-player` (특정 플레이어를 다른 테이블로 이동)
  - 또는 `POST /admin/tables/merge` (소스 테이블 → 타겟 테이블 전원 이동)
  - 핸드 진행 중이 아닌 상태에서만 이동 가능
  - 9명이면 이 항목 불필요

## P1: 버그 / 운영 핵심 (Should Fix)

### 5. 리더보드 칩 이중 계산 버그
- **현상**: `total_chips = account.wallet_balance + table_stack`. 그런데 `wallet_balance`는 이미 테이블 칩을 포함 (`apply_game_delta` 주석 참조). 착석 중인 플레이어 칩이 2배로 표시됨.
- **예시**: 40칩 지급 → 착석 → 리더보드에 80칩으로 표시
- **파일**: `app/services/leaderboard_service.py:49`
- **해결**: `total_chips = account.wallet_balance` (table_stack 더하기 제거)

### 6. close_table 시 칩 이중 지급 버그
- **현상**: `close_table()`이 `transfer_from_table()`을 호출해 seat.stack을 wallet에 더함. 하지만 wallet에 이미 포함되어 있으므로 이중 지급.
- **예시**: wallet=50(테이블 칩 포함), stack=50 → close → wallet=100 (50칩 허공에서 생성)
- **파일**: `app/services/table_service.py:88-89`, `app/services/chip_service.py:92-111`
- **해결**: close_table에서 `transfer_from_table` 호출 제거. wallet은 이미 최신 상태.

### 7. 대회용 셋업 스크립트 없음
- **현상**: `scripts/reset_and_setup.py`가 봇 3테이블 셋업 전용. 인간 전용 대회 셋업 없음.
- **필요 기능**:
  - 기존 테이블/봇 전체 삭제
  - 참가자 수에 따라 테이블 생성 (9명 이하 → 1개, 10명 → 2개)
  - 인간 계정에 칩 40 일괄 지급 (기존 잔액 초기화 후)
- **해결**: `scripts/setup_tournament.py` 신규 작성

### 8. 리더보드 기본 필터가 봇 포함
- **현상**: `/viewer/leaderboard`가 기본으로 봇 포함 (`include_bots=True`).
- **파일**: `app/api/viewer/views.py:185`
- **해결**: 기본값을 `include_bots=False`로 변경
